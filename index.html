🌐 Структура доменного имени

Сегодня я разобрала структуру доменного имени моего сайта. Мой сайт размещён на платформе GitHub Pages, и его адрес выглядит так: https://durmanchic.github.io/medical_student/.

Доменное имя этого адреса — durmanchic.github.io — состоит из нескольких уровней иерархии DNS (системы доменных имён):

  • Корневой уровень (root) — обозначается точкой «.» в конце полного доменного имени (обычно скрыт). Это вершина всей системы.
  • Домен верхнего уровня (TLD — Top-Level Domain) — io. Это национальный домен Британской Индийской Океанической Территории, часто используемый в IT-проектах.
  • Домен второго уровня (SLD — Second-Level Domain) — github. Это основное имя, зарегистрированное компанией GitHub.
  • Поддомен третьего уровня — durmanchic. Это имя моего аккаунта на GitHub, которое автоматически становится поддоменом для моих сайтов через GitHub Pages.

Таким образом, домен читается справа налево: io → github.io → durmanchic.github.io.

Важно: часть адреса после домена — /medical_student/ — это не доменное имя, а путь к конкретному репозиторию (папке) на сервере.
